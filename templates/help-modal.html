Key Merchant Pro Overview

GameVault Key Merchant Pro is a high-performance inventory and sales pipeline management dashboard. It helps you track game activation keys, analyze financial records, monitor stock durations, and manage supplier transactions.

Key Activation States
  • Available: Keys currently active in stock and ready to be sold.
  • Sold: Dispatched keys. These are logged in the Sales Ledger and calculate profit margin.
  • Reserved: Keys held back for customer orders or pending deals.
  • Rejected: Revoked, faulty, or returned keys. Capital cost is tracked but they represent no profit.
Inventory Aging Alerts

To prevent stock depreciation, the app categorizes key storage duration:
Fresh (under 30 days), Aging (30-90 days), Stale (90-180 days), and Very Stale (180+ days).

Catalog Cover Artworks

GameVault fetches game covers dynamically via the CheapShark / Steam API:
- Single Fetch: Clicking the "Fetch Steam" button next to cover art inputs queries artwork links in real-time.
- Bulk Fetcher: Located in the unified Data Tools & Utilities card in Settings. It runs in safe batches of 100 entries at a time to prevent rate-limiting and tab lag, featuring a Steam fallback search via corsproxy.io if CheapShark is blocked.

Dashboard Widget Console

Customize your analytical workflow by transforming the charts layout into an interactive grid of modular widgets.

Interactive Features
  • Drag-and-Drop Order: Click and drag any card header to swap visual positions on the grid. Custom positions are saved instantly.
  • Variable Width Spans: Click the card options menu and select 1, 2, or 3 columns to scale the card width dynamically.
  • Minimize/Expand: Click to collapse the chart body down to a clean header, or click to restore it.

  • 3D Config Card Flip: Click **Configure** in options to rotate the card in 3D. Setup custom timeframe overrides (e.g. Last 30 Days, This Year) or change chart styles on the fly.

Finances View Grid Spanning

All accounting widgets (including Monthly Trend, Performance Averages, Outflow Allocation, Cost & Revenue, and Markup Analysis) are consolidated inside the **Finances** tab. Just like the dashboard, you can click the ellipsis menu on any Finance card header to choose between 1, 2, or 3 columns to lay out your accounting view.

Bulk Spreadsheet Ingestion

You can import huge files directly from Excel/CSV workbooks. The parser reads standard files and updates your lists in batches.

Expected Column Mappings

The importer matches your sheet columns automatically. Make sure your file contains headers similar to:

Excel Header Maps To Notes
Name / Title Game Title Required to group entries
Vendor / Supplier Supplier Source Defaults to "Direct" if empty
Key / Code Activation Key Stored encrypted locally
Cost / Sell Prices Must be positive numbers
Status State "Open" ➔ Available, "Closed" ➔ Sold, "Rejected"

Pro Tip: Use the **Excel/CSV Bulk Import** card in Settings. The importer handles large batches asynchronously, so the browser stays responsive.

Date Formats: The importer parses standard (YYYY-MM-DD) and international (D/M/YYYY or DD-MM-YYYY) formats. Midnight boundaries are timezone-corrected automatically to prevent date shifts.

Duplicates Check: The importer checks all incoming activation keys against your current catalog. Any duplicates found will be reported as a warning alert when the import concludes.

Catalog Cover Artwork Utilities

You can dynamically retrieve cover images for all games in your catalog inside the unified **Data Tools & Utilities** card in Settings:

  • Dual-Source Fallback Pipeline: Queries CheapShark API first. If CheapShark is rate-limited (429) or blocked by your browser's ad-blocker, the utility instantly queries Steam's Store Search API via corsproxy.io.
  • 100-Item Batching Safeguard: Runs up to 100 entries per click to protect API rate limits and keep browser tabs responsive. If you have more than 100 items remaining, simply click Fetch Catalog Images again to process the next batch.
  • Overwrite Filter: Turn on Overwrite existing covers to fetch new Steam covers for all listings. If unchecked, only entries lacking cover images are queried.

Cloud Synchronization & Backups

Secure your database and collaborate across devices by configuring cloud sync services.

Supabase & Firebase Sync

- Real-time mode: All edits (add, edit, sell, delete) instantly update your cloud tables.
- Manual mode: Edits are saved offline. An "Unsynced Changes" badge will display, allowing you to trigger a batch push via the **Synchronize Now** button.

GitHub Repositories Sync

Configure a Personal Access Token (PAT) to commit your database backups directly to your GitHub repo as a JSON file. Use **Auto-Push on Change** to sync silently in the background, or **Auto-Push on Startup** to download the latest state automatically.

Supabase Database Integration Guide

Connecting GameVault to a remote Supabase project enables cross-device synchronization and secures database backups. Follow these simple steps to configure your database:

Step 1: Create a Supabase Project

Sign in to Supabase Console and click New Project. Choose a name, database password, and server region.

Step 2: Initialize Database Tables

Open your project dashboard, navigate to the SQL Editor in the left sidebar, click New Query, paste the initialization script, and click Run. You can copy or download the SQL schema script directly from the Supabase Setup Wizard inside our Settings view.

Step 3: Retrieve and Paste Credentials

Go to project Settings > API. Copy the Project URL and anon (public) Key. Paste them in Settings > Supabase Database panel and click Connect & Sync.

Step 4: Check Table Status

Click Test Schema inside our settings setup card to run remote queries against each table and verify the setup is successful.

Disputes & Refunds Workflow

License keys sold to buyers are sometimes disputed due to issues like duplicate redemption, regional locks, or publisher revocation. GameVault provides a built-in workflow to log these disputes and adjust your financial sheets automatically.

How to Flag a Dispute

Navigate to the Sales Ledger. Next to any transaction, click the amber warning button (). Confirmed flags will highlight the row in warning orange, slash out the selling price, and update the profit to a capital loss.
Pulsing Sidebar Alerts: Active disputes are dynamically tracked on your dashboard. A pulsing red dispute count badge will appear next to the Sales Ledger menu option in your sidebar to alert you of unresolved tickets.

How Math Adjusts for Refunds

To keep accounting accurate, a disputed/refunded key behaves depending on supplier refund status:
- Revenue: Drops to $0 (since the customer was refunded).
- Platform Fees: Drops to $0.
- Absorbed Loss (Supplier Refund: No): Cost stays at the purchase price, and Net Profit drops to negative (-Cost), showing a capital loss.
- Refunded (Supplier Refund: Yes): Cost is treated as $0.00, and Net Profit drops to $0.00 (no net profit loss is incurred).

Resolving a Dispute

If a dispute is resolved (e.g., the customer withdrew the claim or the publisher restored the key), click the green check mark button () on the disputed row. This restores the transaction's original selling price, fee calculations, and profit margins.

Development vs. Production Environments

GameVault features automated environment segregation to isolate local developer trials from live production inventory configurations and database sync channels.

How Environment Switching Works

The application inspects the active browser bar URL hostname on load:
- Development Mode: Triggered on localhost or 127.0.0.1. The amber DEV MODE badge is shown in the top bar.
- Production Mode: Triggered on all other domains (e.g., GitHub Pages, custom hosted domains). The top bar looks clean and standard.

Namespace Virtualization

In Development Mode, storage queries (LocalStorage reads, writes, and deletions) are virtualized:
- Standard gv_ keys are automatically redirected to gv_dev_ keys (e.g., gv_dev_inventory).
- This ensures that local feature testing or mock imports never overwrite, modify, or corrupt your real live production data in the browser cache.

Checking Active Status

Press F12 or right-click and choose Inspect, then view the console. The application logs its current environment status on startup.

Formulas & Business Logic

Review how analytical metrics are calculated inside Key Merchant Pro:

1. Gross Revenue

The sum of listing prices for all keys sold in the active filters period:
Gross Revenue = Sum(Sale Price)

2. Net Net Profit

Calculated by subtracting acquisition costs, platform commission fees, payment processor charges, and other overhead expenses:
Net Profit = Revenue - Cost - Commission Fees - Overhead Expenses

3. Return on Investment (ROI)

Measures the percentage yield relative to capital spent:
ROI % = (Net Profit / Total Expense Costs) * 100

4. Sell-Through Rate (STR)

The velocity proportion of total inventory that has successfully been sold:
STR % = (Total Sold Keys / Total Added Keys) * 100

5. Stock Speed / Shelf-life Aging

Calculated per key as the duration between when a key is added and when it is marked sold:
Days to Sell = Sale Date - Purchase Date

6. Ending Stock Asset Valuation

Calculated on the closing boundaries of any selected timeframe (e.g. month-end or year-end). It counts keys purchased on/before the cutoff that remain unsold or were sold after that cutoff:
Ending Stock Cost Value = Sum(Item Purchase Cost of Open Keys)

7. Lowest Sold Price

Calculates the lowest selling price recorded for any game title across all matching transaction logs. This establishes your historical floor value:
Lowest Sold Price = Min(Sale Price)

8. Micro Visual Margin Bars

Horizontal 3px bars represent ROI and margin proportions visually. Emerald bars stretch right for positive yield values; red bars stretch left for negative yield values, facilitating rapid visual analytics scans.

Action Notification Center

The Action Notification Center keeps a ledger of recent operations performed inside the application. It acts as an audit log to verify that actions were successfully written or synced.

Features & Capabilities:
  • Unread Count Badge: A red badge appears over the bell icon in the top header when there are new events. Opening the dropdown automatically clears the unread count.
  • Display Limit: You can choose to show either the last 5 or 10 notifications using the dropdown selection next to "Clear All" in the notification header.
  • Recent 10 Actions: The underlying history buffer tracks up to the last 10 actions to remain clean, fast, and token-efficient.
  • Timestamps: Each event displays a localized timestamp indicating precisely when the action occurred.
  • Logged Actions: Logs are triggered on key creations, sale recordings, catalog edits, item restorations, bulk adjustments, backup actions, database sync connections, and undo/redo operations.
  • Reload Persistence: Action logs and display preferences are cached in the browser's LocalStorage so they persist if you refresh the page.
Auditing and Maintenance:

Clicking Clear All at the top of the notification card wipes all events from storage.
